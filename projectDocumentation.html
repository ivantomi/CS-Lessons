Step 1: Define the project

Before getting into specific details, it's crucial to decide what you want to do. Project can be virtually anything, from developing an entirely new product, exploring some topic you find interesing or simply improving upon an already existing product/feature. It's a good idea to try and write a short description of the project's purpose and goals as soon as you come up with the idea, just to make sure it feels right.

Things to do:

  • Project Name
  • Project Description
  • Think about other categories in advance (make sure it feels right)

Example:

  • Space Invaders: The Mobile Game
  • This project involves the development of a simple 2D mobile game that allows players control a spaceship to defend against waves of alien invaders.

Step 2: Project Objectives (SMART Goals)

Things to do:

  • Specific - It answers the questions of "what", "why" and "how". Provides a focused target.
  • Measurable - They answer questions of "how much" or "how many" and allow you to track progress and determine whether you're on track to achieve your goal.
  • Achievable (Attainable) - Considers available resources, skills and constraints. It's important to set a goal that is challenging but still feasible.
  • Relevant (Realistic) - Aligns with your broader objectives and means something. Consider that your goal should contribute to your overall intention/mission/vision.
  • Time-Bound - Specifies when the goal should be achieved. Deadline helps time management.

Example:

  • Specific - Create an engaging space-themed shooter game with interactive gameplay.
  • Measurable - Aim for a minimum of 100 unique players within the first three months of release.
  • Attainable - Ensure your team has the required game development expertise and access to necessary tools.
  • Realistic - The objectives align with the goal of creating an entertaining and marketable mobile game.
  • Time-Bound - Complete the game development within six months.

Step 3: Project Background

Things to do:

Identify challenges or issues that your project addresses.

Example:

Challenges: There is a demand for entertaining and nostalgic mobile games, and the market lacks a simple yet engaging space-themed shooter game.

Step 4: Project Scope

Things to do:

Included - List the specific features and components that will be part of the project.

Excluded - Specify any features that will not be part of the project, even though it might make sense to have them. (For example, you are not interested in a complex UI design, so you will use a simple one instead.)

Example:

Included
  • Player spaceship
  • Alien invaders
  • Player controls
  • Score system
  • Game over screen
Excluded
  • Complex UI design
  • Multiplayer
  • Sound effects

Step 5: Stakeholders

Things to do:

Identify the people who are involved in the project (developers, management, end-users...)

Example:

    Project Manager - Overseeing the project and creating documentation

    Game Developers - Coding and development of the game

    Graphic Designers - Designing the game's visuals

Step 6: Constraints

Things to do:

Identify any limitations or restrictions that might affect the project (time, budget, resources, technology...)

Step 7: Use Cases

Things to do:

Define scenarios that describe how the users will interact with your product/project. It's usually best to take a look at what you described in the project scope.

Example:

  • Player starts the game, controls the spaceship and shoots at alien invaders.
  • Player earns points for each alien defeated and can track their high score.

Step 8: Functional Requirements

Things to do:

List the essential functional requirements for your game. They desribe what the system should be able to do and the features that it must deliver. They also describe the system behavior when some inputs are provided by the user.

(Optional) You can separate them into categories - Must have, Should have, Nice to have.

Example:

  • Spaceship controls (up/down/left/right, tilt)
  • Alien invasion waves
  • Score system
  • Game over condition

Step 9: Non-Functional Requirements

Things to do:

Specify the non-functional requirements focusing on performance, security, scalability, usability.

(Optional) Other possible options: reliability, maintainability, availability, compatibility...

Example:

  • Performance - The game must run smoothly on popular mobile devices.
  • Security - The game must not collect any personal data from the user.
  • Usability - The game must be easy to learn and control.
